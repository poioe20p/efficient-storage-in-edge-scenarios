## Tier 1 selective sync — defaults

The Tier 1 thresholds are tuned to the standard `phases.json` workload and live in code, not in `osken-controller.env`:

- `SS_ENABLED` defaults to `1` ([`scaling_config.py`](../../source/sdn_controller/scaling_config.py))
- `TAU_DADOS_MS` defaults to `65` ms ([`edge_server/source/app.py`](../../source/docker/edge_server/source/app.py), [`selective_sync/hotness.py`](../../source/sdn_controller/selective_sync/hotness.py)) — sized to peer-LAN p95 measured on the standard `phases.json` workload (~40 ms baseline, ~75 ms under `cross_region_hotspot` with WAN emulation) so the breach gate fires under hotspot load while staying silent in `baseline`/`local_moderate`. **Requires WAN emulation** on the nat-router; default profile `metro` (`WAN_RTT_MS=10 ms`) — see [Topology — WAN Emulation](topology/topology_overview.md#wan-emulation-inter-lan-latency).
- `SS_MIN_READS_PER_WINDOW` defaults to `14` ([`scaling_config.py`](../../source/sdn_controller/scaling_config.py)) — sized to per-window read counts seen in `cross_region_hotspot` / `storage_stress`.

Override via env only for off-spec experiments. WAN profile knobs (`WAN_RTT_MS`, `WAN_JITTER_MS`, `WAN_LOSS_PCT`, `WAN_RATE_KBIT`) live in [`source/scripts/wan.env`](../../source/scripts/wan.env); re-tune without rebuild via [`source/scripts/tools/wan_set.sh`](../../source/scripts/tools/wan_set.sh).

The coordinator-state PUB sockets bind on `5561`/`5562` (set in [`build_network_setup.sh`](../../source/scripts/build_network_setup.sh)); set `COORDINATOR_STATE_PUB_PORT=0` to disable. The collector subscribes to both via `--lan1-coord-pub` / `--lan2-coord-pub` flags ([`run_experiment.sh`](../../source/scripts/testing/run_experiment.sh)).
