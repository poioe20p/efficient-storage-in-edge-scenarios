Arahunashi2018ImplementationServerLoadBalancingTechniques (reference)
- They do very similar work, like implement load balancing from user to multiple web servers servers,
using round-robin and least bandwith (collected every 14 seconds). Much better for bandwith based.
- They also introduced the OpenLoad tool for measuring response times.

Linn2019ServerLoadBalancingSDN
- Did something similiar using weighted round robin.

3.
(Increasing SDN Network Performance using Load
Balancing Scheme on Web Server)
- Demonstrates, that algorithms based on Least Loaded server outperform in terms of resource usage, Least Connection and Round Robin 
for dynamic load balancing where sessions need to be mantained (web servers) but for content delivery networks Round Robin is better.
- IP Hash for being able to recover a session/connection in case it fals.
- GNU Wget to measure response time

4.
(Using SDN to Enhance Load Balancing in Cloud
Computing: An Overview and Future Directions)
- Advantages of SDN for LB
    - Dynamic Traffic Engineering
    - Centralized Global Decision Making
    - Application and Network Aware LB Decision Making
